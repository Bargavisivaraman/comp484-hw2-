HTML Terms

On this page you’ll find a quick reference to common HTML terms and a table of frequently used definitions.

Attribute
An HTML element property that provides additional information or functionality for an element.
Block-Level Elements
HTML tags that create blocks of content within the document such as headings and paragraphs.
Child Element
Any HTML element that is contained inside another HTML element.
Closing Tag
Appears after an element’s content and indicates the end of an element.
Comment
Reference information added to the code for documentation purposes.
CMS
Short for Content Management System, a system designed to allow users to manage website content.
CSS
Cascading Style Sheets — a presentation language used to control the layout and design of HTML documents.
Doctype
Short for Document Type Declaration. Appears at the top of HTML documents and tells browsers how to interpret the code.
DOM
Short for Document Object Model, a representation of a document that scripts can manipulate.
Elements
HTML elements consist of a tag, content, and optional attributes.
FTP
Short for File Transfer Protocol, used to transfer files to and from servers.
HTTP
Hypertext Transfer Protocol, which controls the formatting and transmission of HTML pages.
Inline-Level Elements
Elements that appear within the normal flow of text, such as links or emphasized text.
JavaScript
A scripting language used to control interactive behavior on web pages.
Metadata
Additional information about the document, typically stored in meta tags.
MIME Type
Short for Multipurpose Internet Mail Extensions, identifies file formats.
Parent Element
An HTML element that contains another element.
Replaced Element
An element whose contents are replaced by an external resource, such as images.
Tag
Markup characters that identify elements in an HTML document.
URL
Short for Uniform Resource Locator, the address of a document or resource on the web.
W3C
The World Wide Web Consortium, the standards body responsible for authoring web standards.
